# 管理後台 Roadmap

## 1. 文件目的

這份文件整理 `NUCOSMOS POS` 管理後台目前已完成的功能，以及接下來建議優先補強的主要系統能力，作為後續排工與驗收依據。

## 2. 已完成的主要功能

- 登入與 JWT 工作階段
- 左側選單拖曳排序
- 左側選單排序同步到後端帳號設定
- 營運總覽與報表中心整併
- 營收趨勢、收款結構、分類分佈、庫存結構等圖表
- 商品管理
- 商品分類管理
- 商品圖片
- 商品配方、包裝設定與商品成本估算
- 商品庫存管理
- 瑕疵庫存管理
- 原料管理
- 包裝管理
- 雙庫存模型
- 盤點單
- 供應商管理
- 採購與進貨
- 補貨建議
- 訂單管理
- 退款回補庫存
- 班次管理
- 裝置管理
- 營運報表與庫存分析

## 3. 第一優先補強

這一階段是從「可用」走向「正式商用」最關鍵的功能。

### 3.1 使用者與權限管理

- 使用者 CRUD
- 角色 CRUD
- 權限矩陣管理
- 角色對頁面 / API / 操作按鈕的授權
- 停用帳號與強制登出

### 3.2 門市管理

- 門市基本資料維護
- 門市營業時間
- 幣別、時區、稅率、服務費設定
- 門市狀態啟用 / 停用
- 多門市權限指派

### 3.3 裝置參數管理

- POS 平板綁定與解除綁定
- 裝置命名規則
- 印表機設定
- 掃碼器 / 刷卡機 / 周邊設備設定
- 裝置健康狀態與最後上線時間

### 3.4 系統設定

- 全域公司資訊
- 發票 / 稅務 / 付款方式設定
- 訂單編號規則
- 預設報表區間
- UI 偏好與通知偏好

### 3.5 操作日誌 / 審計紀錄

- 誰在什麼時間做了什麼事
- 商品、價格、庫存、退款、採購、設定異動紀錄
- 查詢與篩選
- 匯出稽核資料

## 4. 第二優先補強

這一階段主要提升營運效率與日常管理便利性。

### 4.1 價格與促銷管理

- 期間價格
- 門市別價格
- 折扣規則
- 套餐 / 組合價
- 滿額折扣 / 滿件折扣
- 生效時間與失效時間

### 4.2 付款與財務設定

- 付款方式啟用 / 停用
- 現金、刷卡、電子支付配置
- 退款限制規則
- Manager Override 條件
- 班次對帳摘要

### 4.3 匯出與報表下載

- 訂單匯出 CSV / Excel
- 庫存匯出 CSV / Excel
- 採購與供應商對帳匯出
- 報表快照下載

### 4.4 通知中心

- 低庫存通知
- 即將到期批號通知
- 裝置離線通知
- 退款 / 作廢異常通知

## 5. 第三優先補強

這一階段屬於擴充能力與經營分析提升。

### 5.1 會員 / CRM

- 會員列表
- 會員等級
- 消費紀錄
- 點數 / 儲值
- 會員標籤

### 5.2 供應商對帳

- 採購對帳
- 收貨差異追蹤
- 供應商付款狀態
- 月結資料

### 5.3 多門市彙總分析

- 跨門市銷售排行
- 門市毛利比較
- 門市庫存健康度比較
- 門市低庫存與報廢比較

### 5.4 系統維運能力

- 備份 / 還原
- 系統健康檢查頁
- API 連線狀態
- 版本資訊與更新公告

## 6. 建議實作順序

如果以商用上線為目標，建議依序施工：

1. 使用者 / 角色 / 權限管理
2. 門市管理
3. 裝置參數管理
4. 系統設定
5. 操作日誌 / 審計紀錄
6. 價格與促銷管理
7. 匯出與通知中心

## 7. 現階段結論

目前管理後台已具備商品、庫存、採購、訂單、班次、裝置、報表等核心營運功能。  
若要進入正式商用階段，最需要優先補齊的是：

- 使用者與權限管理
- 門市與裝置參數管理
- 系統設定
- 操作日誌 / 審計紀錄

這四塊補完後，整體後台會從「營運功能完整」進一步提升到「管理流程完整」。
