# 會員與收據兌獎規則

## 公開入口

- 客戶公開入口固定為 `https://nucosmos.io/redeem/{token}`
- 不可暴露 `/erp`
- 收據上可列印：
  - `redeemCode`
  - `redeemUrl`
  - QR Code

## 基本流程

1. 客戶每次結帳後拿到收據。
2. 客戶用手機掃描收據上的 QR Code。
3. QR Code 開啟 `nucosmos.io` 的兌獎頁。
4. 客戶輸入姓名與手機號碼後完成兌換。
5. 系統立即顯示本次抽獎結果、中獎獎項或未中獎訊息。

## 兌獎資格

- 訂單狀態不可為 `VOIDED`
- 付款狀態需為：
  - `PAID`
  - `PARTIALLY_REFUNDED`
  - `REFUNDED`
- 每張收據只能成功兌換一次

## 會員綁定規則

- 兌換時必須輸入：
  - 姓名
  - 手機號碼
- 手機號碼正規化規則：
  - `09xxxxxxxx` 轉為 `+8869xxxxxxxx`
  - `886...` 轉為 `+886...`
  - `+886...` 直接使用

## 抽獎規則

- 每次成功兌換都會進行一次抽獎
- 中獎時：
  - 顯示 `恭喜您中獎了！`
  - 顯示中獎獎項名稱
  - 扣減該獎項剩餘數量
- 未中獎時：
  - 顯示 `銘謝惠顧，再接再厲`
  - 顯示未中獎說明
  - 贈送 `1` 點會員點數

## 點數與抵用券規則

- 只有未中獎時會獲得點數
- 每次未中獎獲得 `1` 點
- 每累積 `5` 點，自動發出 `50 元抵用券`
- 抵用券目前為系統自動發放，不需人工建立

## 額外活動提示

兌獎頁需固定顯示以下文案：

`在店內五星好評、分享到任意社群、或分享 LINE 好友後出示給老闆看，可直接獲得 50 元抵用券。`

目前這段為公開活動提示文案，尚未自動核發。

## 獎項管理規則

後台需提供抽獎獎項管理，至少包含：

- 獎項名稱
- 獎項說明
- 中獎機率
- 剩餘數量
- 啟用狀態
- 顯示順序

限制：

- 啟用中的獎項機率總和不可超過 `100%`
- 公開兌獎頁需顯示：
  - 各獎項機率
  - 各獎項剩餘數量

## 已完成範圍

- 收據建立 `redeemCode` / `redeemUrl`
- 公開兌獎頁 `https://nucosmos.io/redeem/{token}`
- 會員姓名與手機綁定
- 抽獎中獎 / 未中獎結果
- 未中獎加點
- 每 5 點自動發 50 元抵用券
- 後台抽獎獎項管理
- 公開頁顯示獎項機率與剩餘數量
