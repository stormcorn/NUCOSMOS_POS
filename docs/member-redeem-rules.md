# 會員與兌獎規則

## 公開入口

- 客戶公開入口固定為 `https://nucosmos.io/redeem/{token}`
- 不可讓客戶看到 `/erp`
- 收據上可印：
  - `redeemCode`
  - `redeemUrl`
  - QR Code

## 會員登入規則

- 會員登入採用手機簡訊認證
- 公開兌獎頁會先引導會員完成手機簡訊登入，再參加抽獎
- 手機驗證沿用 Firebase Phone Auth
- backend 會驗證 Firebase ID token，並建立或更新 `receipt_members`
- 公開會員 session 使用 HttpOnly cookie `nucosmos_member_session`
- session 預設有效期 90 天

## 手機號碼規則

- 會員資料至少包含：
  - 姓名
  - 手機號碼
- 手機號碼正規化：
  - `09xxxxxxxx` 轉成 `+8869xxxxxxxx`
  - `886...` 轉成 `+886...`
  - `+886...` 保持原樣

## 收據兌獎規則

1. 每次結帳拿到收據後，客戶可掃描收據上的 QR Code 參加抽獎。
2. QR Code 會開啟 `nucosmos.io` 的公開兌獎頁。
3. 每張收據只能兌獎一次。
4. 兌獎前需先完成會員手機簡訊登入。
5. 兌獎頁會顯示中獎與否。

## 會員獎勵規則

- 未中獎：
  - 顯示「銘謝惠顧，再接再厲」
  - 獲得 1 點
- 中獎：
  - 顯示「恭喜您中獎了！」
  - 顯示中獎獎品名稱
- 點數規則：
  - 每次未中獎兌換可得 1 點
  - 累積滿 5 點，自動發 50 元抵用券

## 分享提示

兌獎頁固定顯示以下提示文字：

`在店內五星好評、分享到任意社群、或分享 LINE 好友並出示給店主，也可直接獲得 50 元抵用券。`

這段目前為展示提示，尚未自動驗證分享行為。

## 獎項管理規則

- 獎項與中獎機率可在管理後台維護
- 管理欄位包含：
  - 獎項名稱
  - 獎項說明
  - 中獎機率
  - 剩餘數量
  - 啟用狀態
  - 顯示排序
- 公開兌獎頁要顯示：
  - 各獎項機率
  - 各獎項剩餘數量

## 重要實作規則

- 會員登入與兌獎是兩個步驟，但都在同一個公開頁完成
- 新會員首次登入時，需填寫姓名
- 已存在會員再次登入時，可直接用手機簡訊驗證
- 兌獎時若已有會員 session，應直接綁定該會員，不再要求重填姓名與手機
- 不可再把公開客戶流程放在 `/erp`
